OpeatingSystemConcept-Chapter-07

AnswersForCourse
  • Linux/Unix

这是操作系统概念第九版课后部分题目的答案.
多看看，and pass the exam.

0x01

列出三个与计算机系统环境无关的死锁示例。

  • •两辆车从相反方向穿过单车道桥。
  • •一个人走下梯子而另一个人爬上梯子。
  • 两列火车在同一条轨道上相互靠近。

0x02

请考虑以下系统快照：

使用银行家的算法回答以下问题：
a）矩阵需要的内容是什么？
b）系统是否处于安全状态？
c）如果来自进程P1的请求到达（0,4,2,0），请求是否可以立即被授予？

  • a）过程P0到P4的极值分别为（0,0,0,0），（0,7,5,0），（1,0,0,2），（0,0,2， 0）和（0,6,4,2）。
  • b）系统处于安全状态？ 是。 如果Available等于（1,5,2,0），则可以运行进程P0或P3。 进程P3运行后，会释放其资源，从而允许所有其他现有进程运行。
  • c）请求可以立即批准吗？ 这导致可用的值为（1,1,0,0）。 可以完成的一个处理顺序是P0，P2，P3，P1和P4。 因此可以授予请求。

0x03

是否有可能只涉及一个单线程进程的死锁？ 解释你的答案。

  • 是。 如果进程两次获取非递归锁定。

0x04

考虑一个由四个相同类型的资源组成的系统，这四个资源由三个进程共享，每个资源最多需要两个资源。说明系统无死锁。

  • 是的，这个系统没有死锁。
  • 矛盾证明。假设系统已死锁。这意味着每个进程都持有一个资源并等待一个资源。由于有三个进程和四个资源，因此一个进程必须能够获得两个资源。此过程不需要更多资源，因此它将在完成后返回其资源。

0x05

考虑由n个进程共享的相同类型的m个资源组成的系统。 进程一次只能请求或释放一个资源。 如果满足以下两个条件，则表明系统无死锁：
a）每个进程的最大需求是在一个资源和m个资源之间。
b）所有最大需求的总和小于m + n。

假设N =所有需求的总和（i），A =所有分配的总和（i），M =所有Max（i）的总和。
用矛盾来证明。
假设这个系统不是无死锁的。如果存在死锁状态，则A = m，因为只有一种资源以及资源可以一次只请求和释放一个。
从条件b，N + A = M < m + n。所以我们得到N + m < m + n。 所以我们得到N < n。 它表明至少有一个进程i使得需求（i）= 0.（与死锁的必要条件相矛盾）
从条件a，Pi可以释放至少1个资源。
所以现在有n-1个进程共享m个资源，条件a和b仍然存在。 继续论证，没有进程会永久等待，所以没有死锁。

0x06

请考虑以下系统快照：

使用银行家的算法，确定以下每个状态是否不安全。 如果状态是安全的，请说明进程可能完成的顺序。 否则，说明为什么状态不安全。
a. Available = (0, 3, 0, 1)
b. Available = (1, 0, 0, 2)

  • a）不安全。 进程P2，P1和P3能够完成，但没有剩余的进程可以完成。
  • b）安全。 过程P1，P2和P3能够完成。 在此之后，过程P0和P4也能够完成。

0x07

一条单车道桥梁连接北滕布里奇和南滕布里奇的两个佛蒙特州村庄。 两个村庄的农民使用这座桥将产品运送到邻近的城镇。如果北行和南行的农民同时上桥，这座桥可能会陷入死锁。（佛蒙特州的农民很顽固，无法支持。）使用信号量和/或互斥锁，设计一种伪代码算法，防止死锁。 最初，不要担心饥饿（北方农民阻止南行农民使用桥梁的情况，反之亦然）。
