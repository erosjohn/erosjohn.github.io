OpeatingSystemConcept-Chapter-06

AnswersForCourse
  • Linux/Unix

这是操作系统概念第九版课后部分题目的答案.
多看看，and pass the exam.

0x01

解释抢占和非抢占调度之间的区别。

  • 抢占式调度允许进程在执行过程中被中断，将CPU带走并将其分配给另一个进程。
  • 非抢占式调度确保只有当CPU完成当前CPU突发时，进程才会放弃对CPU的控制。
  • （了解发生CPU调度决策的四个条件）

0x02

假设以下过程在指定的时间到达执行。 每个进程都将运行所列的时间。 在回答问题时，使用非抢先式调度，并根据您必须做出决定时所拥有的信息做出所有决策。

a）使用FCFS调度算法的这些过程的平均周转时间是多少？
b）使用SJF调度算法，这些过程的平均周转时间是多少？
c）SJF算法应该可以提高性能，但是请注意我们选择在0时刻运行流程P1，因为我们不知道两个较短的流程会很快到达。计算如果CPU在前1个单元处于空闲状态然后使用SJF调度时的平均周转时间。请记住，进程P1和P2在此空闲时间内正在等待，因此它们的等待时间可能会增加。该算法可以称为预知调度。

  • a. 10.53
  • b. 9.53
  • c. 6.86
    请记住，周转时间是结束时间减去到达时间，因此您必须减去到达时间以计算周转时间。

0x03

假设调度算法（在短期CPU调度级别）支持那些在最近使用最少处理器时间的进程。 为什么这个算法会支持受I / O约束的程序，却又不会永久地饿死CPU绑定的程序？

  • 它有利于I / O绑定程序，因为它们的CPU突发请求相对较短; 但是，受CPU限制的程序不会饿死，因为受I / O限制的程序会相对频繁地放弃CPU以进行I / O操作。

0x04

传统的UNIX调度程序强制执行优先级编号和优先级之间的反向关系：数字越大，优先级越低。 调度程序使用以下函数每秒重新计算一次进程优先级：
优先级=（最近的CPU使用率/ 2）+基数
其中base = 60且最近的CPU使用率是指一个值，该值指示进程使用CPU的频率，因为上次重新计算了优先级。
假设进程P1的最近CPU使用率为40，进程P2为18，而进程P3为10.当重新计算优先级时，这三个进程的新优先级是什么？ 基于此信息，传统的UNIX调度程序是否会提高或降低CPU绑定进程的相对优先级？

  • 分配给流程的优先级分别为80,69和65。 调度程序降低了CPU绑定进程的相对优先级。

0x05

为什么调度程序将I / O绑定程序与CPU绑定程序区分开来很重要？

  • I / O绑定程序具有在执行I / O之前仅执行少量计算的属性。 此类程序通常不会耗尽其整个CPU量。
  • 另一方面，CPU绑定程序使用它们的整个量程而不执行任何阻塞I / O操作。
  • 因此，可以通过为I / O绑定程序提供更高优先级并允许它们在CPU绑定程序之前执行，从而更好地利用计算机资源（CPU和I / O设备）。

0x06

在第5章中，我们讨论了各种内核数据结构的可能竞争条件。大多数调度算法都维护一个运行队列，该队列列出了可以在处理器上运行的进程。 在多核系统上，有两种通用选项：
（1）每个处理核心都有自己的运行队列，或者
（2）所有处理核心共享一个运行队列。每种方法的优点和缺点是什么？

  • 每个处理核心具有自己的运行队列的主要优点是，当调度程序在2个或更多处理器上并发运行时，单个运行队列不会发生争用。 当必须为处理核心做出调度决策时，调度程序只需要查看其私有运行队列。
  • 单个共享运行队列的缺点是必须使用锁来保护它以防止竞争条件和处理核心可用于运行线程，但它必须首先获取锁以从单个队列中检索线程。 但是，负载平衡可能不是单个运行队列的问题，而当每个处理核心都有自己的运行队列时，不同的运行队列之间必须存在某种负载平衡。

0x0A

考虑一个运行十个I / O绑定任务和一个CPU绑定任务的系统。 假设I / O绑定任务为每毫秒CPU计算发出一次I / O操作，并且每个I / O操作需要10毫秒才能完成。 还假设上下文切换开销为0.1毫秒，并且所有进程都是长时间运行的任务。 在以下情况下描述循环调度程序的CPU利用率：
a）时间量是1毫秒
b）时间量是10毫秒

  • a）时间量是1毫秒：无论调度哪个进程，调度程序都会为每个上下文切换产生0.1毫秒的上下文切换成本。 这导致CPU利用率为1 / 1.1 * 100 = 91％。
  • b）时间量是10毫秒：I / O绑定任务在耗尽仅1毫秒的时间量后才会产生上下文切换。 因此，循环所有进程所需的时间是10 1.1 + 10.1（因为每个I / O绑定任务执行1毫秒然后引发上下文切换任务，而CPU绑定任务执行10毫秒之后才会产生 上下文切换）。 因此CPU利用率为20 / 21.1 100 = 94％。
