OpeatingSystemConcept Chapter-01

AnswersForCourse
  • Linux/Unix

这是操作系统概念第九版课后部分题目的答案.
多看看，and pass the exam.

0x01

操作系统三个主要目的

  • 让用户更方便有效的在硬件上运行程序。
  • 根据需要分配计算机的单独资源以解决给定的问题。 分配过程应尽可能公平和有效。
  • 作为控制程序，它有两个主要功能：（1）监督用户程序的执行以防止错误和不正确使用计算机，以及（2）管理I / O设备的操作和控制。

0x02

内核模式和用户模式的区别

  • 只有在CPU处于内核模式时，才能执行某些指令（特权指令）
  • 例如：获得硬件设备，允许/不允许中断。

0x03

以下哪些指令应具有特权

  • 设定定时器的值
  • 清除内存
  • 关闭中断
  • 修改设备状态表中的条目
  • 从用户切换到内核模式
  • 获得IO
    不需要特权的
  • 读时钟
  • 发出陷阱指令

0x04

定时器可用于计算当前时间。 简要介绍如何实现这一目标

  • 程序可以使用以下方法使用计时器中断来计算当前时间。 该程序可以在将来的某个时间设置计时器并进入睡眠状态。 当它被中断唤醒时，它可以更新它的本地状态，它正用于跟踪它迄今为止收到的中断数。 然后，它可以重复此过程，在实际引发中断时不断设置定时器中断并更新其本地状态。

0x05

在多道程序和时间共享环境中，多个用户同时共享系统。 这种情况可能导致各种安全问题？
1.2个问题？

  • 窃取或复制用户的文件
  • 在内存中写入另一个程序（属于另一个用户或OS）区域
  • 使用系统资源（CPU，磁盘空间）没有适当的审计
  • 在打印其他用户的文件时，通过发送数据使打印机混合输出
    2.我们能否在分时机器中确保与专用机器相同的安全性？ 解释你的答案
  • 可能不是，因为人类设计的任何保护方案也可以被打破 - 而且方案越复杂，就越难以对其正确实施充满信心。

0x06

描述对称和非对称多处理之间的差异。 多处理器系统有三个优点和一个缺点？
差异

  • 对称处理将所有处理器视为相等; 可以在任何一个上处理I / O. 可以在其中任何一个上安排流程。
  • 非对称处理指定一个CPU作为主设备，这是唯一能够执行I / O的CPU; 主设备在其他CPU之间分配计算工作。
    优点
  • 多处理器系统可以通过共享电源和外围设备来节省资金
  • 可以更快地执行程序
  • 提高了可靠性
    缺点
  • 多处理器系统在硬件和软件方面都更复杂。 管理协作需要额外的CPU周期，因此每CPU效率会下降

0x07

集群系统与多处理器系统有何不同？ 属于集群的两台机器需要合作才能提供高可用性服务？

  • 集群系统通常通过将多个计算机组合到单个系统中来执行，以执行分布在群集中的计算任务。 另一方面，多处理器系统可以是包括多个CPU的单个物理实体。 集群系统的耦合程度低于多处理器系统。 群集系统使用消息进行通信，而多处理器系统中的处理器可以使用共享内存进行通信。
  • 为了使两台计算机提供高可用性服务，应该复制两台计算机上的状态，并且应该不断更新。 当其中一台机器发生故障时，另一台机器可以接管故障机器的功能。

0x08

中断的目的是什么？ 中断如何与陷阱不同？ 陷阱是否可以由用户程序有意生成？ 如果是这样，为了什么目的？

  • 可以使用中断来指示I / O的完成，以便CPU不必花费周期来轮询设备。
  • 中断是硬件生成的信号，可以改变系统内的流量。 陷阱是软件生成的中断。
  • 陷阱可用于调用呼叫系统调用。执行期间发生异常情况时会引发异常。

0x09

许多SMP系统具有不同级别的缓存; 一个级别是每个处理核心的本地级别，另一个级别在所有处理核心之间共享。 为什么缓存系统是这样设计的？

  • 不同级别基于访问速度和大小。 通常，缓存越靠近CPU，访问速度越快。 但是，更快的缓存通常更昂贵。 因此，较小和较快的高速缓存放置在每个CPU的本地，并且较大但较慢的共享高速缓存在几个不同的处理器之间共享。

0x0A

考虑一个类似于图1.6所示的SMP系统。 举例说明驻留在内存中的数据实际上如何在每个本地缓存中具有不同的值。

  • 假设处理器1将值为5的数据A从主存储器读入其本地高速缓存。 类似地，处理器2也将数据A读入其本地高速缓存。 处理器1然后将A更新为10.然而，由于A驻留在处理器1的本地高速缓存中，因此更新仅在那里发生而不在处理器2的本地高速缓存中。

0x0B

描述与传统PC设计操作系统相比，移动设备设计操作系统的一些挑战。

  • 较少的存储容量意味着操作系统必须仔细管理内存。
  • 操作系统还必须仔细管理功耗。
  • 较低的处理能力和较少的处理器意味着操作系统必须谨慎地将处理器分配给应用程序。

0x0C

定义以下类型操作系统的基本属性

  • Batch:具有类似需求的作业通过操作员或自动作业定序器（非交互式）一起批处理并作为一组通过计算机运行。 通过缓冲，离线操作，假脱机和多道程序设计来保持CPU和I / O设备始终处于忙碌状态，从而提高了性能。
  • Interactive：由许多短交互组成，输入和输出在屏幕上读/写; 下一交互的结果和时间可能无法预测。 注意，纯交互系统（没有分时）只有一个用户; 例如，PC）。
  • Time-sharing:使用CPU调度和多道程序设计来提供系统的经济交互使用。 CPU从一个用户快速切换到另一个用户。
  • Real-time:系统必须在固定的时间内响应输入/命令，以确保正确的性能。 通常从传感器读取输入。
  • Distributed:在几台计算机之间划分计算。 计算机不共享内存或时钟; 他们通过网络互相沟通。
